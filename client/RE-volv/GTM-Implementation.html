Tag Implementation
Spec Sheet

Google Tag Manager configuration notes for campaign measurement, developer implementation, QA, and audit validation.

> Audit Context

Current GTM audit results suggest the campaign measurement foundation is incomplete. The spec sheet prioritizes confirmation of the GA4 base tag, the All Pages trigger, and the attachment between them, then extends into conversion events and UTM variable handling.

Because site access is limited, this document distinguishes between likely real failures and possible misflags so the developer can validate the live state before rebuilding.

01

Required GTM Components

Component Type Name Required State Purpose
Tag GA4 - Base Configuration Active Loads the GA4 measurement ID and sends the baseline page_view event.
Trigger All Pages Active Fires the GA4 base tag on every page load.
Tag-Trigger Link GA4 -> All Pages Linked + Active Ensures the GA4 base tag actually fires on the site.
Tag Conversion Linker Active Preserves Google Ads click attribution for downstream conversion measurement.
Tag Microsoft Clarity - Official Active Supports behavioral review and session replay. Helpful for UX review but not a substitute for GA4 conversion tracking.

03

Audit-Specific Findings

ID: gtm_ga4_base_tag FAIL

The GA4 base tag appears missing from the inspected GTM setup. If true, page-level analytics may not be firing through GTM at all.

ID: gtm_all_pages_trigger FAIL

The standard page-view trigger appears missing. Even if a GA4 tag exists, it will not run unless an equivalent trigger is attached.

ID: gtm_ga4_all_pages_attachment FAIL

The audit could not confirm that the GA4 base tag is linked to the All Pages trigger. This is the last connection in the firing chain.

ID: gtm_conversion_linker PASS

This is good and should remain active. It supports ad click attribution but does not replace GA4 setup.

ID: gtm_clarity_tag PASS

Clarity is active, which suggests GTM itself may be loading. Makes GA4 failures more likely to be genuine configuration gaps.

ID: gtm_site_domain_reference PASS

The workspace appears pointed at a site domain. Lowers chance that audit is looking at unrelated container.

04

Developer Validation Guide

Without direct site access, the cleanest validation path is container-side. A tag can be marked missing for several reasons: it truly doesn't exist, it's only in an unpublished workspace, it was renamed, or it's hardcoded on the site template.
Check What to Inspect Interpretation
Container Workspace Look for a GA4 configuration tag and an All Pages trigger in current workspace. If neither exists, the failures are probably real.
Published Version Open Versions and compare latest published container against workspace. If tag exists only in draft, live audit will still show missing.
Tag Naming Confirm whether an equivalent GA4 base tag exists under a different name. If names differ, the audit may be too strict.
Hardcoded GA4 Ask whether GA4 was placed directly in the site template instead of GTM. If yes, "missing in GTM" is technically true while GA4 works.
Equivalent Trigger Check whether a custom trigger replaces the standard All Pages trigger. If functionally equivalent, audit may be flagging naming.
05

Recommended Build Order

01

Create GA4 Base Tag

The site needs a stable analytics foundation before campaign data can be trusted.

02

Create All Pages Trigger

The GA4 base tag needs a page-load trigger to run sitewide.

03

Attach Tag to Trigger

This closes the base firing chain and ensures activation on load.

04

Publish & Validate

Do not assume a workspace change is live until it is published and tested in DebugView.

05

Conversion Events

Traffic without conversion measurement produces incomplete campaign reporting.

06

Confirm UTM Consistency

Ensure campaign data maps cleanly into source, medium, campaign, content, and term.

06

QA Checklist for Developers

Checkpoint Pass Condition
GA4 base tag exists A GA4 configuration tag is visible in GTM and points to correct measurement ID.
All Pages trigger exists A page-view trigger for all pages exists or equivalent sitewide trigger documented.
GA4 linked to trigger The GA4 base tag has the All Pages trigger attached.
Published version updated The live container version includes the new or corrected setup.
No duplicate base tags Only one intended GA4 base implementation, or duplicates documented.
Conversion events defined Main conversion events and their trigger logic are documented.
UTM variables documented utm_source, utm_medium, utm_campaign, etc. available where needed.
Naming convention documented Campaign naming format is defined and shared with campaign owner.
